# Verification Loop Skill

テスト失敗 → 修正 → 再テストのループを管理するスキル。

## 使用タイミング

- `npm run dev` でエラーが出た時
- コンポーネントが正しく表示されない時
- アニメーションが動作しない時

## ループの流れ

```
1. エラーメッセージを確認
2. 原因を特定（console, devtools）
3. 最小限の修正を実施
4. 再度実行して確認
5. 3回失敗したら停止して報告
```

## 報告フォーマット（3回失敗時）

```
## Verification Failed

- **Error**: エラーメッセージ
- **Attempts**: 試した修正内容
- **Hypothesis**: 原因の仮説
- **Request**: 人間への依頼事項
```

## チェックポイント

- [ ] エラーメッセージを正確に読んだか
- [ ] 変更は最小限か
- [ ] 副作用を考慮したか
